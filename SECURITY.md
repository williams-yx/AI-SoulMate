# Security Policy

感谢你帮助提升 AI SoulMate 项目的安全性。

## Supported Versions

当前仓库处于持续迭代阶段，默认仅保证最新主分支（`master`）与最新发布版本接收安全修复。  
历史分支与过旧版本通常不再单独维护。

## Reporting a Vulnerability

如果你发现了安全漏洞，请不要在公开 Issue 或讨论区直接披露细节。

请通过以下方式私下联系维护者：

- 邮箱：`security@aisoulmate.example`（如未启用，请先通过仓库维护者提供可用安全联系方式）
- 或私信仓库维护者（Gitee / GitHub）

提交时建议包含：

- 漏洞类型与影响范围
- 复现步骤（PoC）
- 受影响文件/接口
- 可能的修复建议（可选）

## Response SLA

我们会尽力按以下目标响应：

- 24-72 小时内：确认收到报告
- 7 天内：完成初步评估并反馈严重等级
- 30 天内：发布修复或缓解方案（复杂问题可能延长）

## Disclosure Policy

在修复发布前，请不要公开漏洞细节。  
我们鼓励在修复上线后进行负责任披露，并在征得同意后致谢报告者。

## Scope

以下内容属于本策略关注范围：

- 认证与授权绕过
- 敏感信息泄露（Token、Key、数据库凭据等）
- 远程代码执行、命令注入、SQL 注入
- 账户接管、越权访问、支付流程安全问题

以下通常不计入漏洞奖励/优先处理：

- 仅影响低风险文本内容的问题
- 无实际可利用性的理论问题
- 第三方服务本身漏洞（非本仓库可控范围）

## Secrets & Configuration

请勿提交任何真实密钥或凭据，包括但不限于：

- API Keys / Access Tokens
- 数据库密码、Redis 密码
- SSH 私钥、证书私钥
- 支付与云服务凭据

请始终使用环境变量与 `.env.example` 模板管理配置。
